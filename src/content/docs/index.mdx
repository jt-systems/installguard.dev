<section class="ig-section not-content" style="margin-top: clamp(4rem, 8vw, 6.5rem);">
  <span class="ig-eyebrow">What it catches</span>
  <h2 class="ig-section-title">Checks mapped to real package attacks.</h2>
  <p class="ig-section-lede">InstallGuard does not guess at a fuzzy package “risk score.” It checks concrete facts about each locked package and lets policy decide whether to allow, warn, or block.</p>
  <div class="ig-grid ig-grid--2">
    <article class="ig-card">
      <div class="ig-card-eyebrow">Identity</div>
      <h3 class="ig-card-title">Typosquats &amp; homoglyph attacks</h3>
      <p class="ig-card-body">Distance-1 typos and visually-confusable Unicode against a curated list of high-traffic packages, with an allowlist for the legitimate distance-1 names that would otherwise create noisy false positives.</p>
    </article>
    <article class="ig-card">
      <div class="ig-card-eyebrow">Provenance</div>
      <h3 class="ig-card-title">Publisher &amp; attestation changes</h3>
      <p class="ig-card-body">Maintainer churn, brand-new publisher accounts, and missing or mismatched attestations across npm <span class="ig-chip">--provenance</span> bundles and PyPI <span class="ig-chip">PEP 740</span> Trusted Publisher releases.</p>
    </article>
    <article class="ig-card">
      <div class="ig-card-eyebrow">Execution</div>
      <h3 class="ig-card-title">Install-time code execution</h3>
      <p class="ig-card-body">Deny-by-default policy on <span class="ig-chip">preinstall</span>, <span class="ig-chip">install</span>, <span class="ig-chip">postinstall</span>, PyPI <span class="ig-chip">setup.py</span>, and in-tree <span class="ig-chip">backend-path</span> build backends, with a curated allowlist for native-binary packages and pattern-based detection of obfuscated payloads across both shell and Python.</p>
    </article>
    <article class="ig-card">
      <div class="ig-card-eyebrow">Release trust</div>
      <h3 class="ig-card-title">Cooling-off windows for new releases</h3>
      <p class="ig-card-body">Configurable minimum release age (default: 24h). Flags new <span class="ig-chip">bin</span> entries or scripts vs. the previous release of the same package.</p>
    </article>
    <article class="ig-card">
      <div class="ig-card-eyebrow">Distribution</div>
      <h3 class="ig-card-title">Dist-tag rollback patterns</h3>
      <p class="ig-card-body">Detects suspicious <span class="ig-chip">latest</span> tag regressions across major versions &mdash; a common signal of compromised maintainer activity or rollback attacks.</p>
    </article>
    <article class="ig-card">
      <div class="ig-card-eyebrow">Disclosed</div>
      <h3 class="ig-card-title">Known advisories &amp; repo health</h3>
      <p class="ig-card-body">OSV / GHSA matching with severity gating, plus deprecation notices, archived upstream repos, and licence-allowlist violations from external package-health signals.</p>
    </article>
  </div>
</section>

<section class="ig-section not-content" style="margin-top: clamp(4rem, 8vw, 6.5rem);">
  <span class="ig-eyebrow">Why it exists</span>
  <h2 class="ig-section-title">Stops installs, not just reports.</h2>
  <p class="ig-section-lede">Most tooling tells you after a lockfile changed. InstallGuard reads the lockfile first and exits before the package manager downloads or runs install-time code. That gives mixed-package-manager teams one policy layer across <code class="ig-chip">npm install</code>, <code class="ig-chip">pnpm install</code>, <code class="ig-chip">yarn install</code>, and <code class="ig-chip">uv sync</code>.</p>
  <div class="ig-grid ig-grid--3">
    <article class="ig-card">
      <div class="ig-stat-num">~2s</div>
      <h3 class="ig-card-title">Fast enough for every PR</h3>
      <p class="ig-card-body">Warm-cache scan of a 1,000-package lockfile in under two seconds. Small enough to run on dependency bumps, branch protection checks, and local preflight installs without becoming “that slow security step.”</p>
    </article>
    <article class="ig-card">
      <div class="ig-stat-num">0</div>
      <h3 class="ig-card-title">Read-only by design</h3>
      <p class="ig-card-body">Reads <span class="ig-chip">package-lock.json</span>, <span class="ig-chip">pnpm-lock.yaml</span>, <span class="ig-chip">yarn.lock</span>, <span class="ig-chip">uv.lock</span>, <span class="ig-chip">poetry.lock</span>, and pinned <span class="ig-chip">requirements.txt</span> directly. Never installs and never executes lifecycle scripts. Network calls (registry metadata, OSV, deps.dev, Scorecard, PyPI Integrity API) are opt-out per provider, and <span class="ig-chip">--frozen</span> runs entirely from <span class="ig-chip">installguard.lock</span> &mdash; zero sockets.</p>
    </article>
    <article class="ig-card">
      <div class="ig-stat-num">100<span style="color:#6ee7b7;font-size:0.7em">%</span></div>
      <h3 class="ig-card-title">Every verdict explains itself</h3>
      <p class="ig-card-body">Every verdict is recorded in <span class="ig-chip">installguard.lock</span> with the exact signals that produced it. Deterministic reruns, JSON and JUnit output, and GitHub PR summaries make it usable in real review and audit workflows, not just local demos.</p>
    </article>
  </div>
</section>
